In Agile methodology, ambiguity is not necessarily something that must be eliminated. However, unclarity, inconsistent features, missing requirements, and duplication are all consequences of ambiguity that must be reduced. The reason for this is that these issues can lead to technical debts which could compromise software quality.

As a result, we proposed the QUAS framework, a set of quality criteria that user stories should meet in order to prevent potential ambiguities. In the QUAS framework, criteria are classified according to the linguistic levels where the potential ambiguity occurs and contra situations of RE problems (i.e., unclarity, inconsistency, inadequacy, duplication). Apart from proposing some criteria, the QUAS framework also proposes recommendations for potential problematic user stories according to the linguistic level at which the problems occur. Consequently, one user story might have multiple linguistic problems and be recommended for improvement in multiple ways.

The criteria in the QUAS framework are depicted as follows:

Req. Quality Linguistic level
Lexical Syntactic Semantic Pragmatic*
Clarity Precise term. Well-formedness Conciseness Well-understood*
Consistency Atomicity Mutual understanding*
Adequacy Conceptually sound Supportive*
Uniqueness Uniqueness

* does not supported by the QUAS prototype

In order to use the QUAS framework as a tool to identify potential ambiguity in user story, each criterion, violation, and improvement was described as follows:

Req. Quality Criterion Violation Recommendation for improvement
Clarity Precise term. Unclear/imprecise term. Replace unclear/imprecise term. with a standard glossary proposed by Müter et al. (2019)
Well-formedness Ill-formedness Change the structure of user story using Connextra template “As a [role], I want [goal], so that [benefit]”
Conciseness Verbosity Detect subordinate conjunction and inquire users to change user story into simple sentence
Consistency Atomicity Complex and/or compound sentence Detect coordinate conjunction and inquire users to change user story into simple sentence
Adequacy Conceptually sound Under and/or over specified Inquire users to provide interpretation of the expected action
Uniqueness Uniqueness Duplication Remove one of two duplicated user stories

In order to facilitate the adoption of the QUAS framework, we developed the QUAS prototype that utilizes several NLP techniques to identify potential ambiguities in user stories.
